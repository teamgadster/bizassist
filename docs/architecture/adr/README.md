# BizAssist ADR System

This directory contains Architecture Decision Records (ADRs) for BizAssist.

## Purpose

ADRs capture architecturally significant decisions, their context, tradeoffs, and consequences.
They are the official architecture memory for the platform.

## Rules

- File format: `ADR-XXXX-title.md`
- Numbering is sequential and immutable.
- ADR records are never deleted.
- Status may change over time.

## Status Lifecycle

- Proposed
- Accepted
- Deprecated
- Superseded
- Rejected

## Workflow

Problem Identified
-> Architecture Discussion
-> ADR Drafted
-> Architecture Review
-> ADR Accepted

## Template

Use `TEMPLATE.md` for all new ADRs.

## Canonical Index

See `ADR-INDEX.md` for the current ADR set.
